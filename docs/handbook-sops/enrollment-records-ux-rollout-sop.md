# SOP: Enrollment Records UX Rollout and Team Enablement

## Scope
Use this SOP to train admins and operations staff on the redesigned Enrollment Records experience.

Covers:
- New Admin Dashboard navigation structure.
- Enrollment Records system model (People view and Groups view).
- Daily usage patterns for each view.
- Team rollout and adoption checkpoints.

## Roles
- Admin
- Super Admin
- Team Leads/Trainers

## Summary of Design Changes
1. Admin navigation is grouped by job-to-be-done:
   - Enrollment Operations
   - Revenue and Performance
   - Platform Admin
2. Enrollment Records is now the single canonical entry point for enrollment work.
3. Enrollment Records has two segmented views:
   - People
   - Groups
4. Both views now share the same top structure:
   - Title and actions
   - People/Groups segmented switch
   - Inline filters
5. Each view has one primary action:
   - People: New Enrollment
   - Groups: Enroll a Group

## Training Script (Use Verbatim)
1. "We now operate from one enrollment system called Enrollment Records."
2. "Inside Enrollment Records, switch between People and Groups using the segmented control."
3. "People is for individual enrollment records. Groups is for employer-group workflows."
4. "Both screens have the same structure so you do not need to relearn controls."
5. "Use the primary button in each view: New Enrollment for People, Enroll a Group for Groups."

## Role-Based Cheat Sheet
### Admin and Super Admin
1. Open Admin Dashboard.
2. Click Enrollment Records under Enrollment Operations.
3. Confirm you land in your last-used view (People or Groups).
4. Apply filters from the inline filter row before taking actions.
5. Use secondary buttons for utility only (for example, Export or Refresh).

### Team Leads and Trainers
1. Demo People first, then Groups.
2. Show where filters moved (inline under the segmented switch).
3. Emphasize one-primary-action rule in each view.
4. Validate each trainee can switch views and complete one task in each.

## Standard Daily Workflow
1. Enter through Admin Dashboard -> Enrollment Records.
2. Choose People or Groups based on task intent.
3. Set filters before opening records.
4. Use primary action if creating new work.
5. Use utility actions (Export/Refresh) only after context is set.

## Validation Checklist
- [ ] User can find Enrollment Records from dashboard in under 10 seconds.
- [ ] User can switch People and Groups without navigation confusion.
- [ ] User can identify primary action in current view.
- [ ] User can apply at least one filter and explain result.
- [ ] User can complete one People task and one Groups task.

## Rollout Plan (1 Week)
### Day 1
- Announce change and distribute this SOP.
- Run 20-minute live walkthrough.

### Day 2-3
- Role-based hands-on practice.
- Collect friction points in a shared log.

### Day 4
- Review friction trends.
- Apply minor copy or label adjustments if needed.

### Day 5
- Confirm completion checklist for all trained users.
- Lock this SOP as current baseline for onboarding.

## Escalation and Feedback
- Product owner: final call on naming and workflow conventions.
- Engineering: fixes for navigation, filters, and action hierarchy.
- Training lead: ownership of onboarding updates.
